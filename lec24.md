# Multi-Programming and Memory

In a **multiprogramming system**, many processes stay in **RAM at the same time**.

Why?

- Keep **CPU utilization high**
- Make system **fast and responsive**
- Run **multiple programs simultaneously**

Because many processes share memory, the **OS must manage memory carefully**.

---

# Memory Isolation & Protection

Each process must run **independently**.

That means:

- One process **cannot access another process's memory**
- One process **cannot modify OS memory**

OS ensures this using:

- Virtual Address Space
- Base Register
- Limit Register
- Memory Management Unit (MMU)

---

## Example of Memory Protection

RAM
OS Memory
Process P1

Process P2


Allowed ranges:


P1 → 1000 to 2000
P2 → 2001 to 3000


If **P2 tries to access 1500**, OS blocks it.

Result:


Memory Protection Fault


This prevents processes from interfering with each other.

# Logical vs Physical Address

## Logical Address

- Generated by **CPU**
- Also called **Virtual Address**
- Used by program
- Range: `0 → max`

Example

logical address =16


---

## Physical Address

- Actual location in **RAM**
- Generated after translation by **MMU**

Formula:

Physical Address = Base + Logical Address


Example


Base = 128
Logical = 16

Physical = 128 + 16 = 144


---

# Memory Protection (Base & Limit Register)

Two registers are used.

### Base Register
- Starting address of process in RAM

Example

Base = 100040


### Limit Register
- Maximum allowed logical address

Example


Limit = 74600


Check performed:


Logical Address < Limit


If true → address valid  
If false → **trap (error)**

# Types of Contiguous Allocation

## 1. Fixed Partitioning

Memory is divided into **fixed partitions**.

Example


Partition 1 → P1
Partition 2 → P2
Partition 3 → P3
Partition 4 → Empty

### Problems

**Internal Fragmentation**

Unused space inside partition.

Example


Partition = 10MB
Process = 7MB
Unused = 3MB


---

**External Fragmentation**

Memory free but **not in one continuous block**.

Example


Free blocks:
2MB + 3MB + 4MB = 9MB

Process needs = 8MB contiguous


Process cannot load.

---

**Low Multiprogramming**

Number of partitions is fixed → fewer processes run.

---

# Dynamic Partitioning

Memory partition created **when process arrives**.

Example

RAM
OS

P1 → 5MB
P2 → 2MB
P3 → 3MB
P4 → 4MB


### Advantages

- No internal fragmentation
- No limit on process size
- Better memory use
- Higher multiprogramming

### Limitation

Still causes **external fragmentation**.

# Quick Summary

| Term | Meaning |
|-----|------|
| Logical Address | Generated by CPU |
| Physical Address | Actual RAM address |
| MMU | Converts logical → physical |
| Base Register | Start of process memory |
| Limit Register | Allowed memory range |
| Contiguous Allocation | One continuous memory block |
| Fixed Partition | Memory divided beforehand |
| Dynamic Partition | Memory allocated when needed |

Memory management allows multiple processes to safely share RAM by translating logical addresses to physical addresses and allocating memory efficiently.
